# 아보카도(AVOCADO) 프론트엔드 종합 기술 보고서

**문서 버전**: v1.0  
**작성일**: 2025년 1월  
**문서 목적**: 본 문서는 아보카도 프로젝트의 프론트엔드 시스템 아키텍처, 기술 스택, 구현 내용을 종합적으로 기술하여 프로젝트 인수인계 및 추가 개발을 위한 기술적 기반을 제공합니다.

---

## 1. 프로젝트 개요

### 1.1 프로젝트 배경 및 목적

아보카도는 AI Voice Communication Development Tool의 약자로, 발화 장애, 성대 질환, 또는 의사소통에 어려움을 겪는 사용자들을 위해 개발된 AI 기반 텍스트-음성 변환 웹 애플리케이션입니다.

프론트엔드는 사용자와 직접 상호작용하는 인터페이스 계층으로서 다음과 같은 역할을 수행합니다. 첫째, 사용자 인증 화면을 제공하여 회원가입과 로그인 프로세스를 처리합니다. 둘째, TTS 변환을 위한 입력 폼과 결과 재생 인터페이스를 제공합니다. 셋째, 커스텀 음성 모델 생성을 위한 녹음 및 업로드 인터페이스를 구현합니다. 넷째, 음성 변환 기록을 조회하고 관리할 수 있는 UI를 제공합니다. 마지막으로 백엔드로부터 받은 데이터를 사용자 친화적인 형태로 시각화합니다.

### 1.2 프론트엔드 핵심 기능

**사용자 인증 인터페이스**는 회원가입과 로그인 화면을 제공합니다. 회원가입 폼은 이메일, 비밀번호, 이름, 생년월일, 성별 입력을 받으며, 실시간 유효성 검사를 통해 입력 오류를 즉시 피드백합니다. 비밀번호는 영문, 숫자, 특수문자를 포함한 10자 이상이어야 하며, 확인 필드를 통해 일치 여부를 검증합니다.

**TTS 변환 인터페이스**는 사용자가 텍스트를 입력하고 음성 모델을 선택할 수 있는 폼을 제공합니다. 커스텀 드롭다운 컴포넌트를 통해 코난, 짱구, 케로로 중 원하는 음성을 선택할 수 있으며, 변환 버튼 클릭 시 로딩 인디케이터가 표시됩니다. 변환이 완료되면 커스텀 오디오 플레이어가 나타나 음성을 재생할 수 있습니다.

**커스텀 오디오 플레이어**는 HTML5 audio 엘리먼트를 기반으로 구현된 맞춤형 컨트롤러입니다. 재생과 일시정지를 토글할 수 있는 버튼, 드래그 가능한 진행 바, 현재 재생 시간과 전체 시간 표시 기능을 제공합니다. 재생이 완료되면 자동으로 초기 상태로 돌아갑니다.

**파일 다운로드 및 공유**는 생성된 음성 파일을 WAV 또는 MP3 형식으로 다운로드할 수 있습니다. 드롭다운 메뉴를 통해 원하는 형식을 선택하면 즉시 다운로드가 시작됩니다. 공유 기능은 모달 창을 통해 카카오톡, 이메일, 인스타그램으로 공유할 수 있는 옵션을 제공합니다.

**마이페이지**는 사용자 정보와 변환 기록을 관리하는 통합 인터페이스입니다. 프로필 사진 업로드, 비밀번호 변경, 변환 기록 조회, 환경 설정 메뉴를 제공하며, 각 섹션은 독립적으로 전환됩니다.

### 1.3 기술 스택

**프레임워크 및 라이브러리**로는 React 19.1.1을 사용합니다. React는 컴포넌트 기반 UI 개발로 재사용성이 높고, 가상 DOM을 통한 효율적인 렌더링을 제공합니다. 단일 HTML 파일 구조로 구현되어 있지만 React의 핵심 개념을 활용하여 상태 관리와 이벤트 처리를 수행합니다.

**빌드 도구**는 Vite 7.1.2를 채택했습니다. Vite는 빠른 개발 서버 시작 속도와 HMR을 제공하여 개발 생산성을 크게 향상시킵니다. 파일 수정 시 즉시 브라우저에 반영되어 실시간으로 결과를 확인할 수 있습니다.

**스타일링**은 Tailwind CSS를 CDN 방식으로 적용했습니다. 유틸리티 우선 CSS 프레임워크로 빠른 UI 구축이 가능하며, CDN 방식을 사용하여 별도 빌드 설정이 불필요합니다. 커스텀 CSS 변수를 통해 브랜드 컬러를 일관되게 관리합니다.

**HTTP 통신**은 브라우저 내장 Fetch API를 사용합니다. 별도 라이브러리 없이 RESTful API 통신이 가능하며, Promise 기반의 비동기 처리를 지원합니다. async/await 패턴을 활용하여 가독성 높은 코드를 작성했습니다.

**아이콘**은 Font Awesome 6.4.2를 사용하여 다양한 시각적 요소를 표현합니다. 마이크, 재생, 다운로드, 공유 등 기능을 직관적으로 나타내는 아이콘을 제공합니다.

**외부 SDK**로는 Kakao SDK 2.7.1을 통합하여 카카오톡 공유 기능을 구현했습니다. JavaScript 키를 사용하여 초기화하고, Share API를 통해 텍스트와 링크를 공유합니다.

## 2. 시스템 아키텍처

### 2.1 프론트엔드 역할 및 책임

프론트엔드는 전체 3계층 아키텍처에서 클라이언트 계층을 담당합니다. 웹 브라우저에서 실행되는 React 애플리케이션으로, 개발 환경에서는 localhost:5173 포트에서 실행됩니다.

프론트엔드의 주요 책임은 다음과 같습니다. 사용자 입력을 받아 유효성을 검증하고, 백엔드 API에 HTTP 요청을 전송합니다. 백엔드로부터 받은 JSON 응답을 파싱하여 화면에 렌더링하고, 에러 발생 시 사용자에게 친화적인 메시지를 표시합니다. JWT 토큰을 localStorage에 저장하고 관리하며, 보호된 API 호출 시 Authorization 헤더에 포함시킵니다.

### 2.2 디렉토리 구조

프론트엔드 프로젝트의 디렉토리 구조는 다음과 같이 구성됩니다.

frontend 루트 디렉토리에는 index.html이 위치합니다. 이 파일은 메인 HTML 파일로 모든 UI 코드가 포함되어 있습니다. HTML 섹션에서는 페이지 구조와 각 화면 섹션을 정의하고, style 태그 내에서 Tailwind CSS와 커스텀 스타일을 작성하며, script 태그 내에서 모든 JavaScript 로직을 구현합니다.

package.json은 프로젝트 메타데이터와 의존성을 관리합니다. react와 react-dom 19.1.1 버전, vite 7.1.2 버전, eslint 관련 패키지들이 정의되어 있습니다.

vite.config.js는 Vite 개발 서버 및 빌드 설정을 정의합니다. React 플러그인을 활성화하고 포트 설정 등을 구성합니다.

eslint.config.js는 코드 품질 규칙을 정의합니다. JavaScript와 JSX 파일에 대한 린팅 규칙을 설정하고, 사용하지 않는 변수에 대한 예외 처리 등을 포함합니다.

node_modules 디렉토리는 npm install 명령으로 설치된 모든 의존성 패키지가 저장되는 곳입니다.

### 2.3 주요 화면 구성

**메인 화면**은 서비스 소개 및 시작 버튼을 제공하는 랜딩 페이지입니다. 아보카도 로고와 프로젝트 개요가 표시되며, 소개 보기와 시작하기 버튼을 통해 다음 단계로 진행할 수 있습니다. 그라데이션 오버레이 효과와 페이드 인 애니메이션으로 시각적 완성도를 높였습니다.

**로그인 화면**은 이메일과 비밀번호 입력 필드를 제공합니다. 비밀번호 필드에는 보기/숨기기 토글 아이콘이 있어 입력 내용을 확인할 수 있습니다. 회원가입 링크를 통해 회원가입 화면으로 전환할 수 있으며, 뒤로가기 버튼으로 메인 화면으로 돌아갈 수 있습니다.

**회원가입 화면**은 이메일, 비밀번호, 비밀번호 확인, 이름, 생년월일, 성별 입력 필드를 제공합니다. 비밀번호는 실시간으로 유효성 검사를 수행하여 조건 충족 여부를 시각적으로 표시합니다. 생년월일은 연, 월, 일을 별도로 입력받으며, 입력이 완료되면 자동으로 다음 필드로 포커스가 이동합니다.

**TTS 변환 화면**은 텍스트 입력창, 음성 모델 선택 드롭다운, 변환 버튼으로 구성됩니다. 변환 결과는 커스텀 오디오 플레이어로 표시되며, 다운로드와 공유 버튼이 함께 제공됩니다. 탭 인터페이스를 통해 내 목소리 모델 섹션과 전환할 수 있습니다.

**내 목소리 모델 화면**은 녹음 시작 버튼과 녹음 상태 표시 영역으로 구성됩니다. 녹음 중에는 빨간 점이 깜빡이며 녹음 중임을 나타냅니다. 현재는 UI만 구현되어 있으며, 실제 녹음 및 모델 생성 기능은 향후 개발 예정입니다.

**마이페이지**는 프로필 이미지, 사용자 정보, 비밀번호 변경, 변환 기록 조회, 환경 설정 메뉴로 구성됩니다. 프로필 이미지는 클릭하여 변경할 수 있으며, 각 메뉴 항목은 독립적인 섹션으로 전환됩니다. 변환 기록은 최신순으로 정렬되어 표시되며, 각 항목마다 오디오 플레이어가 제공됩니다.

## 3. 핵심 기능 구현

### 3.1 인증 시스템

#### 클라이언트 측 인증 흐름

사용자가 로그인 폼에 이메일과 비밀번호를 입력하고 로그인 버튼을 클릭하면, 클라이언트는 입력값의 기본적인 유효성을 검사합니다. 이메일 형식이 올바른지, 비밀번호가 입력되었는지 확인한 후 백엔드 API에 POST 요청을 전송합니다.

백엔드로부터 성공 응답을 받으면 JWT 토큰을 localStorage에 저장합니다. 이 토큰은 이후 모든 보호된 API 요청에서 Authorization 헤더에 포함됩니다. 사용자 이메일도 함께 저장하여 마이페이지에서 표시할 수 있도록 합니다.

로그인이 성공하면 메인 앱 화면으로 전환되고, 실패하면 에러 메시지를 모달 창으로 표시합니다. 토큰이 만료되거나 유효하지 않은 경우 자동으로 로그아웃 처리되어 로그인 화면으로 이동합니다.

#### 회원가입 유효성 검사

회원가입 화면에서는 여러 단계의 유효성 검사가 수행됩니다.

비밀번호 필드는 입력할 때마다 실시간으로 검사됩니다. 정규표현식을 사용하여 영문, 숫자, 특수문자가 각각 하나 이상 포함되어 있고 전체 길이가 10자 이상인지 확인합니다. 조건을 충족하면 녹색 텍스트로 표시되고, 그렇지 않으면 빨간색으로 표시됩니다.

비밀번호 확인 필드는 원본 비밀번호와 일치하는지 실시간으로 비교합니다. 일치하지 않으면 "비밀번호가 일치하지 않습니다" 메시지가 빨간색으로 표시됩니다.

생년월일은 연도 4자리, 월 2자리, 일 2자리를 각각 입력받습니다. 각 필드는 최대 길이에 도달하면 자동으로 다음 필드로 포커스가 이동합니다. 입력이 모두 완료되면 나이가 자동으로 계산되어 숨겨진 필드에 저장됩니다.

성별은 라디오 버튼으로 선택하며, 키보드 탭 네비게이션이 올바르게 작동하도록 구현했습니다.

모든 필드가 유효한 상태에서 회원가입 버튼을 클릭하면 백엔드로 데이터를 전송합니다. 이메일 중복 등의 서버 측 오류는 응답 메시지를 그대로 사용자에게 표시합니다.

#### 비밀번호 재설정

비밀번호 재설정은 두 단계로 진행됩니다.

첫 번째 단계에서는 사용자가 가입 시 사용한 이메일 주소를 입력하고 인증 버튼을 클릭합니다. 백엔드는 해당 이메일로 6자리 인증 코드를 발송하고, 프론트엔드는 "인증 코드가 이메일로 전송되었습니다" 메시지를 표시합니다.

두 번째 단계에서는 인증 코드 입력 필드와 새 비밀번호 필드가 활성화됩니다. 사용자는 이메일로 받은 코드를 입력하고 새로운 비밀번호를 설정합니다. 새 비밀번호도 회원가입과 동일한 유효성 검사를 거칩니다.

모든 정보를 입력하고 비밀번호 변경 버튼을 클릭하면, 백엔드에서 코드를 검증하고 비밀번호를 업데이트합니다. 성공하면 메인 앱 화면으로 이동하고, 실패하면 해당 오류 메시지를 표시합니다.

### 3.2 TTS 변환 인터페이스

#### 음성 모델 선택

음성 모델 선택은 커스텀 드롭다운 컴포넌트로 구현되어 있습니다. 기본 HTML select 요소 대신 div 기반의 커스텀 컴포넌트를 사용하여 더 세밀한 스타일링이 가능합니다.

각 모델은 아이콘과 이름으로 구성됩니다. 코난은 user-secret 아이콘, 짱구는 child 아이콘, 케로로는 frog 아이콘으로 표현됩니다. 드롭다운을 클릭하면 옵션 목록이 펼쳐지고, 원하는 모델을 선택하면 선택된 값이 표시되고 목록이 닫힙니다.

선택된 모델 ID는 숨겨진 input 필드에 저장되어, 변환 요청 시 서버로 전송됩니다. 드롭다운 외부를 클릭하면 자동으로 닫히도록 이벤트 리스너가 등록되어 있습니다.

#### 변환 요청 처리

사용자가 텍스트를 입력하고 음성 모델을 선택한 후 변환 버튼을 클릭하면, 먼저 입력값을 검증합니다. 텍스트가 비어있거나 토큰이 없으면 적절한 오류 메시지를 표시합니다.

검증이 통과되면 로딩 오버레이를 표시하여 사용자에게 처리 중임을 알립니다. 백엔드 API에 POST 요청을 전송하며, Authorization 헤더에 JWT 토큰을 포함시킵니다. 요청 본문에는 변환할 텍스트와 선택한 모델 ID를 JSON 형식으로 담습니다.

백엔드로부터 응답을 받으면 로딩 오버레이를 숨기고, 성공 여부를 확인합니다. 성공한 경우 응답에 포함된 음성 파일 URL을 오디오 엘리먼트의 src에 설정하고, 결과 영역을 화면에 표시합니다. 실패한 경우 에러 메시지를 모달 창으로 표시합니다.

네트워크 오류나 예상치 못한 에러는 try-catch 블록으로 처리되어, 사용자에게 친화적인 오류 메시지를 제공합니다.

### 3.3 커스텀 오디오 플레이어

#### 플레이어 구조

커스텀 오디오 플레이어는 HTML5 audio 엘리먼트를 숨기고, 커스텀 UI 컨트롤을 통해 제어합니다. 플레이어는 재생/일시정지 버튼, 현재 시간 표시, 진행 바, 전체 시간 표시로 구성됩니다.

audio 엘리먼트는 display: none으로 숨겨져 있으며, JavaScript로 제어됩니다. 커스텀 버튼과 진행 바는 Tailwind CSS로 스타일링되어 브랜드 컬러와 일관성을 유지합니다.

#### 재생 제어

재생/일시정지 버튼을 클릭하면 audio 엘리먼트의 paused 상태를 확인합니다. 일시정지 상태이면 play 메서드를 호출하고, 재생 중이면 pause 메서드를 호출합니다.

audio 엘리먼트에서 play 이벤트가 발생하면 버튼 아이콘을 재생에서 일시정지 아이콘으로 변경합니다. pause 이벤트가 발생하면 반대로 일시정지에서 재생 아이콘으로 변경합니다. 이를 통해 현재 상태를 시각적으로 명확하게 표현합니다.

#### 진행 바 제어

audio 엘리먼트의 timeupdate 이벤트는 재생 위치가 변경될 때마다 발생합니다. 이 이벤트 리스너에서 진행 바의 값을 현재 재생 시간으로 업데이트하고, 현재 시간 텍스트도 함께 갱신합니다.

loadedmetadata 이벤트는 오디오 파일의 메타데이터가 로드되었을 때 발생합니다. 이때 진행 바의 최대값을 오디오의 전체 길이로 설정하고, 전체 시간 텍스트를 표시합니다.

사용자가 진행 바를 드래그하거나 클릭하면 input 이벤트가 발생합니다. 이 이벤트 리스너에서 audio 엘리먼트의 currentTime을 진행 바의 값으로 설정하여 재생 위치를 변경합니다.

ended 이벤트는 재생이 완료되었을 때 발생합니다. 이때 진행 바를 0으로 초기화하고, 재생 버튼 아이콘을 원래대로 되돌립니다.

#### 시간 표시 형식

시간은 분:초 형식으로 표시됩니다. formatTime 함수는 초 단위 숫자를 받아 "00:00" 형식의 문자열로 변환합니다. Math.floor를 사용하여 소수점 이하를 버리고, padStart 메서드로 항상 두 자리 숫자를 유지합니다.

### 3.4 파일 다운로드 및 공유

#### 다운로드 기능

다운로드 버튼을 클릭하면 드롭다운 메뉴가 나타나 WAV 또는 MP3 형식을 선택할 수 있습니다. 각 옵션은 anchor 태그로 구현되어 있으며, href 속성에 음성 파일 URL을 설정하고 download 속성에 파일명을 지정합니다.

사용자가 형식을 선택하면 브라우저의 기본 다운로드 동작이 실행됩니다. 파일명에는 현재 타임스탬프를 포함하여 중복을 방지합니다. 다운로드가 시작되면 드롭다운은 자동으로 닫힙니다.

드롭다운 외부를 클릭하면 메뉴가 닫히도록 document 레벨의 이벤트 리스너가 등록되어 있습니다. 이는 사용자 경험을 개선하는 세부적인 처리입니다.

#### 공유 모달

공유 버튼을 클릭하면 모달 창이 나타나 세 가지 공유 옵션을 제공합니다. 모달은 fixed 포지셔닝으로 화면 중앙에 표시되며, 반투명 배경 오버레이를 통해 컨텐츠에 집중할 수 있도록 합니다.

각 공유 옵션은 아이콘과 레이블로 구성되어 있습니다. 인스타그램은 그라데이션 텍스트로, 카카오톡은 브랜드 옐로우 컬러로, 이메일은 회색으로 표현되어 각 플랫폼의 아이덴티티를 반영합니다.

모달 닫기는 X 버튼 클릭, 배경 클릭, ESC 키 입력으로 가능합니다. 이를 통해 다양한 사용자 인터랙션 패턴을 지원합니다.

#### 카카오톡 공유

카카오톡 공유는 Kakao SDK를 사용하여 구현됩니다. 애플리케이션이 로드될 때 JavaScript 키로 SDK를 초기화합니다. 초기화에 실패하면 콘솔에 에러를 기록하고, 공유 시도 시 적절한 메시지를 표시합니다.

사용자가 카카오톡 옵션을 선택하면 Kakao.Share.sendDefault 메서드를 호출합니다. objectType은 text로 설정하고, text 필드에 변환한 텍스트를 포함한 메시지를 작성합니다. link 필드에는 현재 페이지 URL을 설정하여 수신자가 클릭 시 아보카도 웹사이트로 이동할 수 있도록 합니다.

buttons 배열에는 행동 유도 버튼을 정의합니다. "아보카도에서 직접 들어보기" 텍스트와 링크를 설정하여, 수신자가 직접 서비스를 체험할 수 있도록 유도합니다.

#### 이메일 공유

이메일 공유는 mailto 프로토콜을 활용합니다. 사용자가 이메일 옵션을 선택하면 이메일 주소 입력 모달이 나타납니다. 기본값으로 현재 로그인한 사용자의 이메일이 자동 입력됩니다.

수신자 이메일을 입력하고 전송 버튼을 클릭하면, 제목과 본문이 인코딩되어 mailto URL이 생성됩니다.
## 4. 백엔드 API 명세

본 장에서는 프론트엔드에서 호출하는 백엔드 API 엔드포인트와 데이터 구조를 정의하며, 모든 API 통신은 `http://localhost:3001/api`를 기본 URL로 사용합니다. 인증이 필요한 요청에는 **Authorization 헤더**에 **JWT(Bearer Token)**를 포함해야 합니다.

---

### 4.1 인증(Authentication) 관련 API

#### 4.1.1 회원가입 (`POST /api/register`)

* **설명**: 사용자 계정을 생성합니다.
* **인증 필요**: 없음.
* **요청 데이터 (JSON)**:
    * `email` (필수, String): 사용자 이메일 (ID 역할), 이메일 형식 및 중복 불가.
    * [cite_start]`pw` (필수, String): 사용자 비밀번호, 영문, 숫자, 특수문자를 포함한 10자 이상이어야 합니다[cite: 3].
    * `name` (필수, String): 사용자 이름, 100자 이하.
    * `age` (필수, Number): 사용자 나이, 만 나이 기준 정수.
    * `gender` (필수, String): 성별, `male` 또는 `female`.
* **성공 응답 (HTTP 201 Created)**: `{"success": true, "message": "회원가입이 성공적으로 완료되었습니다."}`.
* **오류 응답 (HTTP 400 Bad Request)**: `{"success": false, "message": "이미 사용 중인 이메일 주소입니다."}`.

#### 4.1.2 로그인 (`POST /api/login`)

* **설명**: 사용자를 인증하고 JWT를 발급받습니다.
* **인증 필요**: 없음.
* **요청 데이터 (JSON)**:
    * `email` (필수, String): 사용자 이메일.
    * `pw` (필수, String): 사용자 비밀번호.
* **성공 응답 (HTTP 200 OK)**: `{"success": true, "token": "...", "message": "로그인 성공"}`.

#### 4.1.3 비밀번호 재설정 관련 API

[cite_start]프론트엔드에서는 인증 코드가 **123456**으로 하드코딩된 임시 메시지로 표시됩니다[cite: 3]. 실제 구현 시 백엔드는 다음 두 엔드포인트를 분리하거나 하나의 통합 엔드포인트로 처리해야 합니다.

* **인증 코드 전송 (`POST /api/auth/send-code`)**:
    * **설명**: 비밀번호 재설정을 위해 이메일로 인증 코드를 전송합니다.
    * **요청 데이터 (JSON)**: `{"email": "사용자_이메일"}`
* **비밀번호 재설정 (`POST /api/auth/reset-password`)**:
    * **설명**: 인증 코드를 확인하고 새 비밀번호로 업데이트합니다.
    * **요청 데이터 (JSON)**: `{"email": "사용자_이메일", "code": "6자리_인증코드", "newPassword": "새_비밀번호"}`

---

### 4.2 TTS 변환 및 조회 관련 API

#### 4.2.1 음성 변환 요청 (`POST /api/convert`)

* **설명**: 텍스트를 AI 음성으로 변환하고 파일을 서버에 저장합니다.
* **인증 필요**: 필요 (JWT).
* **요청 데이터 (JSON)**:
    * `text` (필수, String): 변환할 텍스트 내용.
    * [cite_start]`modelId` (필수, String): 사용할 음성 모델 ID (예: `conan`, `jjanggu`, `keroro`)[cite: 3].
* **성공 응답 (HTTP 200 OK)**:
    * `success`: `true`
    * `message`: "음성 변환 성공"
    * `url`: 변환된 음성 파일에 직접 접근 가능한 URL (예: `http://localhost:3001/storage/tts-results/...`).

#### 4.2.2 변환 기록 조회 (`GET /api/history`)

* **설명**: 로그인한 사용자의 모든 변환 기록을 최신순으로 조회합니다.
* **인증 필요**: 필요 (JWT).
* **성공 응답 (HTTP 200 OK) (JSON 배열)**:
    * [cite_start]각 기록은 다음 필드를 포함합니다: `id` (Number), `text` (String, 변환된 텍스트), `modelId` (String), `fileUrl` (String, 음성 파일 URL), `createdAt` (String, 변환 시각, ISO 8601 형식)[cite: 3].

---

### 4.3 기타 기능 관련 API

#### 4.3.1 프로필 이미지 업로드 (`POST /api/user/profile-image`)

* [cite_start]**설명**: 프로필 이미지를 서버에 저장하고, 저장된 이미지의 URL을 업데이트합니다[cite: 3].
* **인증 필요**: 필요 (JWT).
* **요청 데이터**: `multipart/form-data` 형식으로 `profileImage` 필드에 이미지 파일을 담아 전송합니다.
* **성공 응답 (HTTP 200 OK)**: `{"success": true, "message": "프로필 이미지가 성공적으로 업데이트되었습니다.", "imageUrl": "저장된_이미지_URL"}`.

#### 4.3.2 이메일 파일 전송 (백엔드 구현 시) (`POST /api/share/email`)

현재 프론트엔드는 `mailto` 프로토콜을 사용하고 있으나, 백엔드에서 직접 이메일 전송 기능이 구현될 경우를 대비한 명세입니다.

* **설명**: 생성된 음성 파일을 첨부하거나 다운로드 링크를 포함하여 수신자 이메일로 전송합니다.
* **인증 필요**: 필요 (JWT).
* **요청 데이터 (JSON)**:
    * `recipientEmail` (필수, String): 파일을 받을 수신자의 이메일 주소.
    * `fileUrl` (필수, String): 공유할 음성 파일의 공용 URL.
    * `text` (필수, String): 공유 메시지에 포함할 원본 텍스트.

## 5. 배포 및 환경 설정

### 5.1 프론트엔드 환경 및 빌드

[cite_start]프론트엔드 개발은 **Vite 7.1.2**를 빌드 도구로 사용합니다[cite: 3]. 개발 생산성을 위해 빠른 개발 서버 시작 속도와 HMR(Hot Module Replacement)을 제공합니다.

* **개발 서버 실행**: `npm run dev` 명령어로 `http://localhost:5173`에서 서버를 실행합니다.
* **프로덕션 빌드**: `npm run build` 명령어로 배포 가능한 정적 파일이 `dist` 디렉토리에 생성됩니다. [cite_start]`.gitignore` 파일에는 이 `dist` 디렉토리가 포함되어 있어 버전 관리에서 제외됩니다[cite: 1].
* **의존성**: **React 19.1.1** 및 `@vitejs/plugin-react` 플러그인을 사용합니다.

### 5.2 코드 품질 관리 (ESLint 및 Prettier)

[cite_start]코드의 일관성과 품질 관리를 위해 **ESLint**와 **Prettier**가 설정되어 있습니다[cite: 2].

* **ESLint 설정 (`eslint.config.js`)**:
    * [cite_start]**플러그인**: `@eslint/js`, `eslint-plugin-react-hooks`, `eslint-plugin-react-refresh`를 사용합니다[cite: 3].
    * [cite_start]**규칙**: 사용하지 않는 변수(`no-unused-vars`)에 대해 에러를 발생시키며, 대문자 시작 변수는 무시하도록 예외 처리되어 있습니다[cite: 3].
    * [cite_start]**무시 패턴**: 빌드 결과물인 `dist` 디렉토리를 린팅 대상에서 제외합니다[cite: 3].
* **Prettier 설정 (`.prettierrc`)**:
    * [cite_start]코드 포맷팅을 위해 **printWidth** 80, **tabWidth** 2, **singleQuote** `true`, **trailingComma** `all`, **semi** `true`, **useTabs** `false`가 설정되어 있습니다[cite: 4].
    * [cite_start]`eslint-config-prettier` 패키지를 통해 ESLint의 포맷팅 규칙이 Prettier와 충돌하지 않도록 비활성화되었습니다[cite: 2].

### 5.3 백엔드 인수인계를 위한 참고 사항

백엔드 개발 팀은 제공된 **설치 및 실행 가이드 (`setup.md`)**를 참고하여 서버를 구성해야 합니다.

* **백엔드 기본 환경**: Node.js와 MySQL 8.0 이상이 필요합니다.
* **환경 변수**: `.env` 파일에 `PORT=3001`, `DB_HOST`, `DB_PASSWORD`, `JWT_SECRET` 등의 중요 설정값을 포함해야 합니다.
* **저장소 경로**: 생성된 음성 파일은 백엔드의 `storage/` 디렉토리에 저장되어야 하며, `http://localhost:3001/storage`를 통해 접근 가능해야 합니다.
* **프론트엔드 연결**: 현재 프론트엔드가 하드코딩한 `http://localhost:3001` 엔드포인트에 맞춰 백엔드 서버를 포트 **3001**에서 실행해야 합니다.
* **API 경로 접두사**: 모든 비즈니스 로직 API는 `/api` 접두사를 사용합니다 (예: `/api/login`).
